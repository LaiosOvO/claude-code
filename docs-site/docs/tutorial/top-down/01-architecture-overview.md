# 第一章：全局架构鸟瞰

> 本章从最高层视角理解 Claude Code 的整体设计，让你在脑中建立一张完整的地图。

## 1.1 Claude Code 是什么？

Claude Code 是 Anthropic 官方的 **CLI AI 编程助手**。它不是简单的命令行工具——它是一个完整的：

- **终端 UI 应用**（用 React + Ink 实现终端界面）
- **AI Agent 框架**（支持工具调用、子 Agent、多轮对话）
- **远程协作平台**（支持远程控制、多会话通信、跨机器迁移）

```
│              服务层                           │

├─────────────────────────────────────────────┤
│              基础设施层                       │
│  Bun Runtime / WebSocket / File I/O / Git    │
└─────────────────────────────────────────────┘
```

| 类别 | 技术 | 为什么选它 |
|------|------|-----------|
| 运行时 | Bun | 比 Node.js 更快的启动、原生 TS 支持、内置 bundler |
| 语言 | TypeScript | 类型安全，大型项目必备 |
| 终端 UI | React + Ink | 用 React 组件模型构建终端界面，声明式 UI |
| CLI 解析 | Commander.js | 成熟的 CLI 参数解析库 |
| API 客户端 | Anthropic SDK | 官方 SDK，类型完整 |
| 协议 | MCP (Model Context Protocol) | AI 工具标准协议 |

## 1.4 核心概念地图

在深入代码之前，先理解这几个核心概念：

### Tool（工具）
Claude 调用的能力。比如 Bash 执行命令、Read 读文件、Edit 编辑文件。每个 Tool 都有：
- **名称** + **输入 Schema** + **执行函数**
- **权限检查** — 是否允许执行
- **并发安全标记** — 能否与其他 Tool 并行
- **UI 渲染** — 在终端中如何显示

### QueryEngine（查询引擎）
对话的核心。它：
1. 收集系统提示词 + 用户消息 + 上下文
2. 调用 Claude API
3. 解析响应中的工具调用
4. 执行工具，收集结果
5. 把结果反馈给 Claude，循环直到结束

### Command（命令）
用户用 `/xxx` 调用的功能。比如 `/commit`、`/help`。

### Skill（技能）
可扩展的命令系统。用户可以在 `.claude/skills/` 放 Markdown 文件来定义新技能。

### Bridge（桥接）
远程控制系统。让外部（手机、网页）能控制本地的 Claude Code。

### Session（会话）
一次对话。包含消息历史、上下文、工具状态。

## 1.6 数据流总览

```
用户输入 "帮我写一个函数"
    │
    ▼
┌─ REPL Screen ──────────────────────────────┐
│  解析输入 → 检查是否是命令 (/xxx)             │
│  如果是普通消息 → 提交给 QueryEngine          │
└────────────┬───────────────────────────────┘
             │
             ▼
┌─ QueryEngine ──────────────────────────────┐
│  1. 组装系统提示词                           │
│     - 默认系统提示                           │
│     - CLAUDE.md 文件内容                     │
│     - 工具描述                               │
│     - Git 状态                               │
│  2. 构建消息数组 [system, ...history, user]   │
│  3. 调用 Claude API (流式)                   │
│  4. 解析响应                                 │
│     - 文本块 → 直接显示                       │
│     - 工具调用块 → 进入工具执行               │
└────────────┬───────────────────────────────┘
             │
        ┌────┴────────────┐
        ▼                 ▼
   文本响应            工具调用
   (直接显示)      ┌────┴────┐
                   ▼         ▼
              权限检查    多个工具
              (allow?     并行执行
               deny?     (如果安全)
               ask?)
                   │
                   ▼
              工具结果
              收集完毕
                   │
                   ▼
            追加到消息历史
            继续循环 → API
```
