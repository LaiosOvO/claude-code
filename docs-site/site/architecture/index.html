    架构文档¶

    本节提供 Claude Code 系统级别的架构分析，包括整体架构设计、数据流、组件关系以及关键设计决策的解读。

    整体架构¶

    Claude Code 是一个基于 TypeScript 构建的 AI 编程助手，运行在终端环境中。其整体架构可以分为以下几个层次：

    graph TB

        CLI[CLI 入口]
        TUI[Ink TUI 界面]
    end

    subgraph 应用层
        CMD[命令处理]
        COORD[多 Agent 协调器]
        QUERY[查询引擎]
    end

    subgraph 核心层
        TOOLS[工具系统]
        CTX[上下文系统]
        STATE[状态管理]

    subgraph 服务层

        AUTH[认证服务]

    end

    COORD --> QUERY

    TOOLS --> PERM
    TOOLS --> MCP

    核心架构特征¶

    1. Ink/React 驱动的终端 UI¶

    Claude Code 使用 Ink（终端版 React）来构建交互界面。这意味着整个 TUI 采用了声明式的组件模型，状态变化自动驱动界面重新渲染。

    2. 插件化工具系统¶

    工具（Tool）是 Claude Code 与外部世界交互的核心抽象。每个工具：

    • 有独立的定义文件
    • 声明输入参数的 Schema
    • 定义权限需求
    • 实现具体的执行逻辑

    3. 多 Agent 协作¶

    通过 Coordinator 模块，Claude Code 支持将复杂任务拆分为子任务，分发给多个 Agent 并行处理。

    4. MCP 协议扩展¶

    通过 Model Context Protocol（MCP），用户可以接入外部工具服务器，扩展 Claude Code 的能力边界。
